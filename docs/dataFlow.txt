====================================================================
DATA FLOW DIAGRAM - RL-Powered Automated Healthcare Assistant
Team Vector | Meta PyTorch OpenEnv Hackathon
====================================================================

[STEP 1] USER INPUT
-------------------------------------------
Patient submits query: "I have chest pain and difficulty breathing"
                    │
                    ▼

[STEP 2] OPENENV ENVIRONMENT (reset)
-------------------------------------------
Environment loads query + patient context
Returns OBSERVATION to Agent
                    │
                    ▼

[STEP 3] RL AGENT DECISION (Ankit)
-------------------------------------------
Agent processes observation via Policy Network
Chooses ACTION:
  - ESCALATE (send to doctor)
  - ANSWER (generate response via LLM)
                    │
                    ▼

[STEP 4] ACTION EXECUTION
-------------------------------------------
IF ESCALATE:
    → Trigger alert system
    → Reward: +10 if correct emergency, -5 if false alarm

IF ANSWER:
    → Send query to Gemini LLM (Shubham)
    → LLM generates response with medical system prompt
    → Response returned to user
                    │
                    ▼

[STEP 5] REWARD CALCULATION (Ritu)
-------------------------------------------
Reward = Base_Reward + Quality_Bonus - Penalties

Base_Reward:
  +10 → Correct escalation
  -20 → Missed emergency (ANSWER when should ESCALATE)
  -5  → False alarm (ESCALATE when not needed)

Quality_Bonus (0 to +5):
  - Relevance to query
  - Medical accuracy
  - Safety compliance

Penalties:
  -0.1 per second of delay
  -2 for unsafe content
                    │
                    ▼

[STEP 6] LEARNING LOOP
-------------------------------------------
Agent receives reward
Agent updates policy (PPO/GRPO)
Environment moves to next state
Repeat until episode ends (done = True)
